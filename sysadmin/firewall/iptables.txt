===========
-0- general
===========
iptables are used to filter network packets.
This can be used to modify or disallow etc.

The commands are iptables and ip6tables

terms:
table: like a database table, contains chains.
chain: list of rules which can match a set of packets.
rules:
	A rule consists of a "packet criteria" and a "target".
	If a packet matches the criteria, then "target" specifies the next rule
	Otherwise, check the next rule in the chain
target
	Can be:
		user-defined chain
		iptables-extensions
		ACCEPT (allow)
		DROP (disallow)
		RETURN (stop this chain and resume at next rule in previous chain)

=============
-0.1- options
=============
[-t|--table] <table>

================
-0.1.1- commands
================
Only 1 command can be given.

[-A|--append] chain rule
	Add one or more rules to end of a chain.
	Rules will be created for all combinations of source/dest

[-C|--check] chain rule
	check if rule exists (indicated by exit code)

[-D|--delete] chain rule[num]
	Delete the rule from the chain.  Num starts with 1.

[-I|--insert] chain [num=1] rule
	Insert rule into the chain.

[-R|--replace] chain num rule
	Replace rule in the chain

[-L|--list] [chain=allchains]
	list rules in the chain.
	Recommended to use with -n argument (reduce many DNS lookups)

[-S|--list-rules] [chain=allchains]
	print all rules in selected chain
	"like iptables-save"

[-F|--flush] [chain=all]
	"This is equivalent to deleting all the rules one by one."
	??"Flush" selected chain??
[-Z|--zero] [chain=all [rulenum=all]]
	zero the packet and byte counters in chosen chains
	add -L to see the state before zeroing.
[-N|--new-chain] chain
	Add new user-defined chain. Name must be unique
[-X|--delete-chain] [chain=all]
	delete user-defined chain.  Must be unreferenced and empty to be deleted.
[-P|--policy] chain [ACCEPT|DROP]
	Set policy for built-in chain.
[-E|--rename-chain] old-chain new-chain
	rename user-defined chain. "cosmetic only."

==================
-0.1.2- parameters
==================
These specify rules etc.

Generally, '!' can be added to invert matches
(ex. for -p, -s, -d, -i, -o...)


-4|--ipv4: ignore ipv4 rules inserted with ip6tables-restore
-6|--ipv6: ignore ipv6 rules inserted with iptables-restore

-p|--protocol [tcp|udp|udplite|icmp|icmpv6|esp|ah|sctp|mh|all]
	Specify the protocol

-s|--src|--source address[/mask][,...]
-d|--dst|--destination address[/mask][,...]
	comma-delimited addresses to use as source.
	Address can be network name, hostname, ip
	mask: number(ex. 24) or mask (ex. 255.255.255.0)
	Multiple addresses will expand to multiple rules

-m|--match <match>
	specify a match to use (extension module that tests for properties.)
-j|--jump <target>
	jump target (what to do when packet match)
	a user-specified rule or special, [ACCEPT|RETURN|DROP], or extension
	No target = do nothing except increment rule counter
	?After continue current rule after completion?
	(man doesn't mention this, but the description for goto seems to imply this.)

-g|--goto <chain>
	like jump, except do not continue processing this chain afterwards.

-i|--in-interface <name>[+]
-o|--out-interface <name>[+]
	name of input/output interface.
	Ending with '+' acts similarly to fnmatch with *
	ex: eth+ matches eth0, eth1, ethanything, ...

-f|--fragment
	Only apply to fragmented packets.

==========
-1- tables
==========
There are 5 tables which might or might not be available
------
filter
------
The default table.
builtin chains:
	INPUT: to local socket
	FORWARD: being routed through current machine (?eg mobile hotspot?)
	OUTPUT: locally-generated packets

---
nat
---
used for new connections
builtin chains:
	PREROUTING
		modify incoming packets
	INPUT
		modify incoming packets bound for localhost
	OUTPUT
		modify locally-generated packets (before routing)
	POSTROUTING
		modify locally-generated packets just before leaving

------
mangle
------
specialized packet modification
builtins chains:
	PREROUTING
	OUTPUT
	INPUT
	FORWARD
	POSTROUTING

---
raw
---
exemptions from connection tracking (NOTRACK target)
builtin chains:
	PREROUTING
	OUTPUT

--------
security
--------
MAC (Mandatory Access Control) rules (SECMARK, CONNSECMARK)
builtin chains:
	INPUT
	OUTPUT
	FORWARD

-2- tools

====================
-2.1- iptables-apply
====================
"safer way to update tables remotely"
New rules might cut connection.
Add a timeout after update prompting user to verify if ok.
If timed out, assume connection was cut (bad rules) and revert.

-t|--timeout <seconds>
-w|--write [savefile=/etc/network/iptables.up.rules]
-c|--command <runcmd>
-h|--help
-V|--version

===================
-2.2- iptables-save
===================
Save rules set to file.

-M|--modprobe prog
-f|--file fname
	output to save to, else write to stdout
-c|--counters
	include counters inoutput
-t|--table <table>
	only save given table.

======================
-2.3- iptables-restore
======================
Load from a save file.
-c|--counters
	restore counters too
-h|--help
-n|--noflush
	don't flush previous table
-t|--test
	dry run
-v|--verbose
-V|--version
-w|--wait seconds
	wait for lock
-W|--wait-interval <msec>
-M|--modprobe prog
-T|--table name
	only resture this table.

=========================
-2.4- iptables-extensions
=========================
These are "extensions" used to match/filter packets/connections.

==========================
-2.4.1- notable extensions
==========================
addrtype
	matches based on address type
		UNSPEC
		UNICSAT
		LOCAL
		BROADCAST
		ANYCAST
		MULTICAST
		BLACKHOLE
		UNREACHABLE
		PROHIBIT
		THROW
		NAT
		XRESOLVE
	--src-type
	--dst-type
	--limit-iface-in
	--limit-iface-out

comment
	--comment <comment>
	Add comments to any rules

connbytes
	Match based on number of transferred packets so far
	--connbytes low[:high]
		Specify range (or !range)
	--connbytes-dir [original|reply|both]
	--connbytes-mode [packets|bytes|avgpkt]
		Specify units for connbytes

iprange
	match range of ip addresses
	--src-range from[-to]
	--dst-range from[-to]

multiport
	Match source/destination ports
	-p must be [tcp|udp|udplite|dccp|sctp]
	--source-ports|--sports [port|port:port][,...]
	--destination-ports|--dports [port|port:port][,...]
	--ports [port|port:port][,...]
	up to 15 port allowed (a:b counts as 2 ports even though is a range.)

pkttype
	--pkt-type [!][unicast|boradcast|multicast]

recent
	Create list of ip addrs
	"example: create a badguy list of people"
	--name name
		list to use for names
	--set
		add source addr to list
	--rsource
		match/save source of each packet in recent list table
	--rdest
		match/save dest of each packet
	--mask netmask
	--rcheck
		check if source addr is in list
	--update
		like --rcheck, except update "last seen" timestamp
	--remove
		remove from list
	--seconds seconds
		seen within last number of seconds
	--reap
		delete entries older than --seconds
	--hitcounts hits
		only match items in list with packets exceeding hits
	--rttl
		addr is in the list and packet matches packets which hit --set rule.
		"This may be useful if you have problems with people faking their source address
		in order to DoS you"
		(?spoofed the address, but packet contents are identical?)

	Lists are stored in /proc/net/xt_recent/*

set
	Match ip sets defined by `ipset`
	--match-set setname flag[,flag...]
	--return-nomatch
	--update-counters
	--update-subcounters
	--[packets|bytes]-[eq|lt|gt] value

socket
	?matches if there is a listening socket?
	--transparent
	--nowildcard
	--restore-skmark

tcp
	--source-port|--sport port[:port]
	--destination-port|--dport port[:port]
	--tcp-flags mask comp
	--syn
	--tcp-option number

time
	filter packet by time
	--datestart YYYY[-MM[-DD[Thh[:mm[:ss]]]]]
	--datestop YYYY[-MM[-DD[Thh[:mm[:ss]]]]]
	--timestart hh:mm[:ss]
	--timestop hh:mm[:ss]
	--monthdays day,...
	--weekdays day,...
	--contiguous
	--kerneltz

==================
-3- best practices
==================
Save rules with iptables-save
This way can have a "checkpoint" to load verified working rule states.

example from:
	https://github.com/paulRbr/minimal-iptables/blob/master/iptables.up.client.rules 

# Generated by iptables-save v1.4.14 on Mon Nov 17 18:45:27 2014
*filter
:INPUT DROP [0:0]
:FORWARD DROP [0:0]
:OUTPUT ACCEPT [0:0]
:LOGGING - [0:0]
# Communicate with established connections tcp and udp
-A INPUT -p tcp -m tcp -m state --state ESTABLISHED -j ACCEPT
-A INPUT -p udp -m udp -m state --state ESTABLISHED -j ACCEPT
-A INPUT -p icmp -m icmp --icmp-type 8 -j ACCEPT
-A INPUT -p icmp -m icmp --icmp-type 0 -j ACCEPT
-A INPUT -i lo -j ACCEPT
-A INPUT -j LOGGING
-A LOGGING -m limit --limit 2/min -j LOG --log-prefix "IPTables Packet Dropped: " --log-level 7
-A LOGGING -j DROP
COMMIT
