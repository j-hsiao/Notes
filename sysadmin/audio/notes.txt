=========
-1- pulse
=========
pulse operates similar to X.
There is a server, clients connect to server to play sound.

sinks: outputs (speakers)
sources: inputs (microphone)

Configs are at:
	/etc/pulse/default.pa
	~/.config/pulse/default.pa


---------------------
environment variables
---------------------
PULSE_SERVER server connection string
PULSE_SINK name of sink to use
PULSE_SOURCE name of source to use
PULSE_BINARY path of pulseaudio exe
PULSE_CLIENTCONFIG path for a config file

---------------------
Recording pulse audio
---------------------
paactl list short sources
number  name    otherdata info stuff...

name is probably something like
alsa_[input|output].pci-XXXXX....analog-stero[.monitor]

To record, pick one of the "output" ones

----------------
recording source
----------------
pactl load-module module-null-sink sink_name=myrecordingsourcename

pactl load-module module-virtual-source source_name=my_record_source \
	master=myrecordingsourcename.monitor

------------------------
-1.1.1.1- device modules
------------------------
All device modules have some common arguments:
	sink_name source_name
		[a-zA-Z0-9._]\{1,128}
	format=s16
	rate=44100
	channels=2
	channel_map: A list of comma-separated channel names. The currently
		defined channel names are: left, right, mono, center, front-left,
		front-right, front-center, rear-center, rear-left, rear-right, lfe,
		subwoofer, front-left-of-center, front-right-of-center, side-left,
		side-right, aux0, aux1 to aux15, top-center, top-front-left,
		top-front-right, top-front-center, top-rear-left, top-rear-right,
		top-rear-center, (Default depends on the number of channels and the
		driver)
	sink_properties,source_properties (0.9.16+)
		additional properties
		same as update-sink-proplist command after sink/source setup

============
-2- pipewire
============
pipewire is successor to pulse audio

configs in
	/etc/pipewire/
	/usr/share/pipewire/
	~/.config/pipewire/pipewire-pulse.conf.d

place in a file in ~/.config/pipewire/pipewire-pulse.conf.d/
	pulse.properties = {
		server.address = [
			"unix:native"
			"tcp:127.0.0.1:4713"
		]
	}

	pulse.cmd = [
		{ cmd = "load-module" args = "module-null-sink sink_name=foo" flags = [ ] }
	]

Have environment variable:
PULSE_SERVER=tcp:127.0.0.1:4713
to use the pipewire-pulse server.
