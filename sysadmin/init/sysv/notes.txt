============
-1- SysVinit
============
This is the old init system.
This splits system initialization into separate run levels:

	0     poweroff.target
	1     rescue.target
	2,3,4 multi-user.target
	5     graphical.target
	6     reboot.target

The `runlevel` prints the previous and current runlevel.

On changes to the runlevel, corresponding scripts are run.

-2- service
===========
service servicename command
	Search for a script in /etc/init.d matching
	servicename or a systemd unit.
	(systemd takes priority)

service --status-all
	Run status on all service scripts found.
	List simple status and service
		+ running
		- stopped
		? no status command
	(This only applies to sysV services.)

NOTE:
	service is now usually a wrapper around all
	the service management frameworks:
		sysV, systemd, etc

	Example:
		service ssh status
		systemctl status sshd
