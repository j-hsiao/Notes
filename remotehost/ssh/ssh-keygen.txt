ssh-keygen tool for ssh keys

NOTE: the "certificates" made by ssh-keygen are only good for sshing.
they are NOT the same as the certificates created by openssl

===========
-1- options
===========
==================
-1.1- key creation
==================
-t type
	key type:
		dsa
		ecdsa
		ecdsa-sk
		ed25519
		ed25519-sk
		rsa
	signature type (rsa):
		ssh-rsa
		rsa-sha2-256
		rsa-sha2-512 (default)

-b bits
	rsa: > 1024, default 3072
	DSA: ==1024
	ECDSA: 256|384|521
	No bit lengths:
		ECDSA-SK
		ED25519
		ED25519-SK
-Z cipher
	Cipher to use for private key
		3des-cbc
		aes128-cbc
		aes192-cbc
		aes256-cbc
		aes128-ctr
		aes192-ctr
		aes256-ctr (default)
		aes128-gcm@openssh.com
		aes256-gcm@openssh.com
		chacha20-poly1305@openssh.com
======================
-1.2- key modification
======================
-f keyfile
	When generating a key, this is the private key.
	The public key will be keyfile.pub
-p
	change password
-P old_passphrase
-N new_passphrase
-i
	"import": read unencrypted private/public keyfile in -m format
	and print openssh private/public key to stdout
-e
	"export": Print keyfile (pub or priv) in given format (-m)
-m <format>
	specify key generation format
	create PEM instead
	formats:
		RFC4716
		PKCS8
		PEM
-C comment
	add comment
-c
	change comment
-y
	read private openssh and print public key to stdout

-1.3- signing
=============
-h
	make a host cert instead of user cert
-I
	an ID
	creating a cert: a (human readable) name for the cert
	match/verify: signer identity

-n principal1,principal2,...
	Specify principals (user/hostnames) for cert
-O option
	options for action.  multiple -O for multiple options
	CERTIFICATES
		TODO: CERTIFICATES section options
		hashalg=alg
		print-pubkey
		verify-time=timestamp
	MODULI GENERATION
	FIDO
		application
		challenge=path
		device
		no-touch-required
		resident
		user
		verify-required
		write-attestation=path
-s ca_key
-U
	CA key is in a ssh-agent (provide the public key to -s)
-V validity-interval
	[start=now:]end
	start:
		always
		YYYYMMDD[HHMM[SS]]
		-N[smhdw]: see -3.1- time values
	end:
		forever
		YYYYMMDD[HHMM[SS]]
		+N[smhdw]: see -3.1- time values
-Y command
	find-principals
		print -s principals found in -f file to stdou
	match-principals
		find principals matching -I key in the -f file
	check-novalidate
		Validate a ?signed? "message"
		message: read from stdin
		-n signature namespace
		-s signature
	sign
		Produce "signature" files for each file in arguments.
		-f: private key to use (if public key, search ssh-agent for the private key)
		-n: signature namespace
			example: "email", "file", generally should be "namespace@your.domain"
	verify
		verify a signature and message.
		stdin: the message file given to -Y sign
		-n signature namespace match -Y sign
		-s "signature" file produced by -Y sign
		-I signer name: match name in allowed signers
		-f allowed signers: see -4-
		-r revoked keys file
-z serial
	specify serial number for signing, prefix with + to increment

================
-2- certificates
================
Certificates consist of
* public key
* identity info
* 1 or more principal names (user/host)
* options signed by a CA key

The purpose of a cert is:
"This cert declares that CA has verified that this public key belongs to this identity."
The cert would be used in place of the public key when sshing

Certificates can have different formats:
	OpenSSH
	X.509

----------------
Creating a cert:
----------------
ssh-keygen [-h] -s /path/to/ca_key -I IdForCert \
	[-n user1,user2...|host1,host2...] \
	[-V start:end] \
	/path/to/key.pub
result: /path/to/key-cert.pub

-------
options
-------
user options:
	clear
	critical:name=contents
	extension:name=contents
	force-command=command
	[no|permit]-agent-forwarding
	[no|permit]-port-forwarding
	[no|permit]-pty
	[no|permit]-user-rc
	[no|permit]-x11-forwarding
	no-touch-required
	source-address=addrlist
	verify-required

no host cert options exist

-3- KRL
=======
Binary files of revoked keys.
ssh-key -k [krlspecfile|pubkey]...

-------
krlspec
-------
serial: serial_number-serial_number
id: key_id
key: public_key
sha1: public_key
sha256: public_key
hash: fingerprint


================
-4- verification
================
ssh-keygen can "sign" a file and verify its contents.

-------
signing
-------
	ssh-keygen -Y sign -f privatekey -n sigspace f1 f2 f3...
	output is f1.sig f2.sig f3.sig ...

---------
verifying
---------
Create an "allowed signers" file:
	principals options keytype b64key

	principals: name of the signer

	options: comma-delimited list of options
		cert-authority
		namespaces=namespace-list
		valid-after=YYYYMMDDHHMM[SS]
		valid-before=YYYYMMDDHHMM[SS]

ssh-keygen -Y verify -n sigspace -s f1.sig -I principal -f allowed_signers < f1
success if verified, else falied
