	-0.0- observations
	==================
	typing in vim, esc to exit insert mode, then <pre> ends up in a raw <pre> being
		sent. It is not interpreted as the tmux <pre> key...

	tmux vim colors are messed up, but if `set default-terminal screen-256color`
		then vim colors are ok.  Without this, the cygwin colors still work,
		but the wsl colors do not.

	=================
	-0.1- terminology
	=================
	Session
		A collection of windows.

	Window
		a "space" that takes the entire "screen"
		The terminal can only display a single window at a time.

	Pane
		A splitting of a "Window"

	Key sequence
		A sequence of keys that generally activate a command
		binding.  Key sequences will be separated by a space.
		A dash will indicate combination with a modifier key.
		Example: "C-b [" means press control+b and then [.

	Prefix key
		All tmux keybindings are prefixed with the "prefix key".
		In these notes, the prefix key will be omitted unless otherwise
		stated referred to as <pre>

	C-x/M-x
		C-x means press Control + the x character.
		M-x means either pressing alt + x, or pressing Esc followed by x.
		Both seem to activate the M-x bindings.

		-0.3.1- variables
		=================
		#{varname}
		#{optname}
		#S where S is short alias
		#{?variable,yes,no}
			it MUST be a variable name, no values allowed.
		#{cmd/opts/...:arg1[,arg2...]}
			binary cmd:
				Direct string comparison:
					< <= > >= == != 
					result: 0/1 depending on comparison
				strmatch
					m/opts
						r: regex (instead of globbing match)
						i: ignorecase
					result: 0/1 depending on match or not
				bool operators:
					|| &&
					YES, these ARE BINARY, if you try to add more than 2, then the extra
					will be tacked on as values for the 2nd value meaning it will always be truthy.
					result: 0/1 depending on any or both

				math operators:
				e|op|opts:v1,v2
					math expr
					NOTE: use | instead of / (?because / is division?)
					op: +-*/% (need to %% if also strftimed)
					opts:
						f: use float
						D: evaluate to D digits
						N: number of decimal places

			unary cmd
				C/opts:
					search for regexp, evaluate to lineno if found else empty
				a
					Convert number to ascii
				=
					Truncate a variable (must be a variable name)
					N: size limit of N chars.  If negative, then take chars from the end
					presuff: use this as prefix/suffix if trimmed.
				p
					Pad a string
					N: Pad to this size, >0 pads left, <0 pads right
				n
					length (bytes)
				w
					length (display)
				t/opts
					convert time to string
					opts:
						f/strformat: format with strftime, need to escape %%
						p: shorter
				b
					basename
				d
					dirname
				q/opts
					quoted
					opts:
						h escape hash characters
				E
					expand twice
				T
					same as E ?but also expand strftime specifiers?
				S
				W
				P
					loop over each session/window/pane, insert the format once for each.
					If multiple args, 2nd is used for current window or active pane.
				N/opts
					check if name exists
					w: window (default)
					s: session
				s/pat/repl/: 
					regex substitution
					opt: i: ignorecase
				l
					literal, do not expand

		-0.3.2- command substitution
		============================
		#(command)
		NOTE: unlike bash, command substitution in tmux only uses the LATEST line.
		where latest means temporally latest.  On the first invocation, there have
		been no lines.  It would very likely be substituted as "<'command arg...' not ready>"
		However, any formatting afterwards would then update to whatever the latest line
		is.  Even if you use display-message -d 0 ..., the formatting happens before
		waiting for keypress, so it will still end up as "not ready".

============
-1- commands
============
Tmux implements its own scripting language which is very similar to bash.
Note that when calling tmux commands from the shell, the shell first parses
arguments, and then tmux.  This means that to chain tmux commands, the semicolon
must be escaped.

If there is no command, then new-session is used by default.
Some commands have aliases.

____________
same as bash
	Words are split by whitespace.
	The first word is a command, the rest are arguments.
	semicolon/new-line terminates a command.
	Quoting is done with '', ""
	"" allows substitutions, '' does not.
	# starts a comment
	Line continuation with trailing \

______________
diff from bash
	"{} is parsed as config file and then converted to string and can be nested."
	What this means is that {} is kind of like using C preprocessor.
	{
		%if ...
		command1
		%else
		command2
		%endif
	}
	will either become 'command1' or 'command2' the %ifs etc are discarded/evaluated
	at the time of evaluating {}.

	commands separated by a ; is more like bash &&
	Line continuation is allowed everywhere except in {}

	?substitutions are only inside double quotes? double quotes are like merging fo bash "" and $'' syntax?
		Get environment variable with $env_var
		~user is replaced with their home dir, defaulting to current user
		\uXXXX \uXXXXXXXX for hex unicode
		\e \r \n \t  for esc, carriage return, newline, tab
		\ooo for octal
		\C literal C character for any beside above mentioned ones.

====================================
[-t|-s] [client|session|window|pane]
	These specify target client/session/window/pane for a command
	______
	client
		The pty (/dev/...) of any currently attached client.
		In the shell, `tmux list-clients` or `tty`. NOTE list-clients only shows
		currently attached client sessions.
	_______
	session
		Checked in order:
			=exactmatch
			$ID
			session-name
			start of session-name
			glob pattern
		Error if ambiguous
	______
	window
		session:window where session is as above, and window is (in order):
			=exactmatch
			special token
			window index
			window id @id
			exact window name
			window name prefix
			glob pattern
		empty indicates next unused index or current window depending on the command.
		______________
		special tokens
			long            meaning
			{start} (^)     lowest window index
			{end} ($)       highest window index
	____
	pane
		window[.id] where window is as above.  Empty id means the active pane.
		______________
		special tokens
			{top}             The top pane
			{bottom}          The bottom pane
			{left}            The leftmost pane
			{right}           The rightmost pane
			{top-left}        The top-left pane
			{top-right}       The top-right pane
			{bottom-left}     The bottom-left pane
			{bottom-right}    The bottom-right pane
			{up-of}           The pane above the active pane
			{down-of}         The pane below the active pane
			{left-of}         The pane to the left of the active pane
			{right-of}        The pane to the right of the active pane
	_____________________
	common special tokens
		{last} (!)        the previously selected item by time
		{next} (+)        the next item by index
		{previous} (-)    the previous window by index
			+- can be followed by a count
		{mouse} (=)       item with the most recent mouse activity
		{marked} (~)      marked item
	________
	item IDs
		$session
		@window
		%pane

	Uncrecognized keys are sent as a string of characters
	-l disable name lookup
	-H hex
	-R ?reset terminal state?
	-M pass through a mouse event
	-X send command into copy mode
	-N repeat count
	-F expand formats

==================
-2.0- observations
==================
_________________________
normal vs repeat bindings
normal bindings: consume keys, do nothing if no binding (consume the keypress)
repeat bindings: check keys, let the keypress go if NO REPEAT BINDINGS IN THE CURRENT TABLE.

	example:
	bind u "something; switch-client -Ttesting"
	This is a normal keybinding that ends with changing to a different key-table.
	If a keypress is done that does not match any in the key-table, then it will be consumed and
	nothing happens.

	bind -r u "something; switch-client -Ttesting"
	This is a repeat keybinding that ends with changing to a different key-table.
	If a keypress is done that does not match any in the key-table, then it will NOT be consumed
	and instead be sent to whatever program is in the current pane.

____________________________
prefix key special treatment
1. bindings to prefix will NEVER be used EVEN IF THE BINDING IS -Troot.
   (except in the prefix table)
2. prefix key after a repeat binding changes to the prefix table if not
   already BUT MAINTAINS REPEAT MODE.  Only repeat bindings are able to be
   activated until repeatmode ends. NOTE it also does not seem to change the
   current repeat-time timeout.
   ex:
   1. set repeat-time to 10000 (10 seconds)
   2. In a repeat keybinding, set repeat-time to 100
   3. press prefix after 5 seconds
   4. within 5 seconds, press another repeat keybinding -> activates
   5. after 5 secnods, press another repeat keybinding -> does NOT activate

Allow prefix to behave like any other keybind:
1. set-option prefix None
2. add a root binding `bind -Troot prefix "switch-client -Tprefix"`

now "prefix" still activates bindings etc
BUT it nolonger takes the weird priority it seems to take if activated via the prefix setting.

-2.7- mouse
===========
mouse bindings are off by default. Set the session option to On to use them.
Mouse events consist of an event and a location
events:
	WheelUp       WheelDown
	MouseDown1    MouseUp1      MouseDrag1   MouseDragEnd1
	MouseDown2    MouseUp2      MouseDrag2   MouseDragEnd2
	MouseDown3    MouseUp3      MouseDrag3   MouseDragEnd3
	SecondClick1  SecondClick2  SecondClick3
	DoubleClick1  DoubleClick2  DoubleClick3
	TripleClick1  TripleClick2  TripleClick3
locations:
	Pane
	Border
	Status
	StatusLeft
	StatusRight
	StatusDefault

example:
	MouseUp1Pane: left clicked on a pane.

default bindings let mouse select/resize panes and copy text.

========================
-2.8- multi-key bindings
========================
tmux key-tables only support a single key for bindings.
However, you can have multiple key tables.
To achieve the effect of multi-key bindings, set a key-table
that activates another keytable with the bindings you want.

example:
	bind-key -Tthirdkeys c dosomething
	bind-key -Tsecondkeys b switch-client -Tthirdkeys
	bind-key -Troot a switch-client -Tsecondkeys

	a -> activates secondkeys (for 1 key)
	b -> activates thirdkeys (for 1 key)
	c -> does the action

The config file is ".tmux.conf" in the user's home directory.

NOTE: On startup, there is no tmux session.  As a result,
any `set-option` command in the conf has no session to set the
options on.  As a result, -g is basically always required
when using `set-option` inside the config file.

==============
-3.1- settings
==============
Settings are set with `set-option` and displayed with `show-options`.
They are usually referred to as "options", but these are separate from the
command-line options in -0.2-.

Generally, there are global and local sets of settings. Panes and windows
?have the same options?, but pane values take priority and are otherwise
inherited from the window.

User options are prefixed with '@'
