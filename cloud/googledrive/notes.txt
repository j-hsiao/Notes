===============
-0.1- resources
===============
The base uri for resources is:
https://www.googleapis.com/drive/v[2|3]

Files should be given a `name` with extension, but when getting the file,
the extension is in a separate `fileExtension` property

----------
File types
----------
https://developers.google.com/workspace/drive/api/guides/mime-types
Besides the typical mime types, google drive also supports these:

"application/vnd.google-apps.*"
* is replaced with:
	audio
	video
	shortcut: google drive shortcut
	folder: google drive folder
	file: google drive file
	spreadsheet: google sheets
	document: google docs
	presentation: google slides
	vid: google vids


--------
metadata
--------
https://developers.google.com/workspace/drive/api/reference/rest/v3/files
NOTE: "string int" means int as a string, ?because json only supports ints up 53 bits?

_____________________
list default metadata
	kind: output only string "drive#file"
	id: string id, can use "root" for MyDrive.
	name: string name, retains the file extension, the "fileExtension" metadata
	      is just splitext (but does not include the dot.)
	mimeType: string mimetype
	resourceKey: key for accessing item via shared link

______________
other metadata
	driveId: readonly string id for shared drives
	fileExtension: readonly string
	size: string int
	parents: list of parent ids, though only 1 parent is supported
	         ? did they intend multiple parents = acts like a hard link?
	         superceded by shortcuts?
	shared: bool
	shortcutDetails: {
		targetId: string id of target
		targetMimeType: output only string mimetype
		targetResourceKey: output only
	}

============
google drive scopes look like:

	https://www.googleapis.com/auth/*

drive[.readonly]            Your entire drive.
drive.metadata[.readonly]   File metadata (but not contents) of your drive
drive.file                  Files created by this app.
drive.appdata               "app configuration data"
drive.apps.readonly         View your Google Drive apps??
drive.meet.readonly         Google Meet files
drive.photos.readonly       Google photos
drive.scripts               Modify your Google Apps Script scripts' behavior

-2.3- resumable
===============
___________________________________________________________________
1. Send the initial request and retrieve the resumable session URI.

	POST https://www.googleapis.com/upload/drive/v3/files?uploadType=resumable

	headers:
		_____________________
		X-Upload-Content-Type
			Optional. Set to the MIME type of the file data, which is transferred in
			subsequent requests. If the MIME type of the data is not specified in the
			metadata or through this header, the object is served as
			application/octet-stream.
		_______________________
		X-Upload-Content-Length
			Optional. Set to the number of bytes of file data, which is transferred in
			subsequent requests.
		____________
		Content-Type
			Required if you have metadata for the file. Set to
			application/json; charset=UTF-8.
		______________
		Content-Length
			Required unless you use chunked transfer encoding. Set to the number of
			bytes in the body of this initial request.

	response has a "Location" header
	example:
		HTTP/1.1 200 OK
		Location: https://www.googleapis.com/upload/drive/v3/files?uploadType=resumable&upload_id=xa298sd_sdlkj2
		Content-Length: 0

	metadata goes in the request body, if any.
____________________________________________
2. Upload the data and monitor upload state.
	PUT to the Location value (the resumable session uri)
	set content-length, file contents in body.
____________________________________________________________
3. (optional) If the upload is disturbed, resume the upload.
	a. get status:
		PUT Locationuri
		Content-Range: */totalbytes or */*
		content-length: 0

		response:
		200 OK / 201 Created -> completed
		404 not found -> session expired
		308 resume incomplete -> must continue
			Check the Range header for bytes received.
			If absent, assume 0.
			example:
				start-: start to implicit end
				start-stop: a range
				-stop, from the end
				start-stop, start-stop,...: multi range

			note: byte range is 0-indexed and inclusive on both ends.

	b. send data
		PUT ...
		Content-Range: bytes start-stop

		send bytes, 0-indexed

============
-3- download
============
GET https://www.googleapis.com/drive/v3/files

------------------
general parameters
------------------
fields: list of fields to use
	default: kind,id,name,mimeType,resourceKey
	comma separated list
	use slash for nesting
	name(fields)
		? not very clear, does this mean
		for every object in the "name" attribute of the returned
		json object, use the requested fields?
	* for all fields
fileId: id of file to get, omit to list files instead.
alt:
	media: include file contents in body
	otherwise, return a "files" resource
acknowledgeAbuse:
	true: acknowledge risk of malware or ?abusive? files

----------------
query parameters
----------------
q: Search for specific values:
	([not] term operator values) [[and|or] ...]

	terms: a file field
		https://developers.google.com/workspace/drive/api/guides/ref-search-terms#file-properties
		lhs
			name contains|[!]=
			fullText contains      searches name, description, indexableText
			mimeType contains|[!]=
			modifiedTime <!=>
			viewedByMeTime <!=>
			trashed [!]= true|false
			starred [!]= true|false
			sharedWithMe [!]= true|false
			createdTime <!=>
			properties has
			appProperties has
			visibility [!]=
			shortcutDetails.targetId [!]=

			hidden [!]= true|false
			memberCount <!=> num
			organizerCount <!=> number
			orgUnitId [!]=
		rhs
			in parents
			in owners
			in writers
			in readers

	operator: condition
		https://developers.google.com/workspace/drive/api/guides/ref-search-terms#operators
		[!]=
		<[=] >[=]
		contains
			name: prefix only
			fullText: anywhere, underscores treated as spaces
		in
		and
		or
		not
		has
	values: values to use
		'single-quoted string, escape \\ and single-quote'
			if spaces in the string, then it should be double-quoted:
			'"a string with spaces"'
		'YYYY-mm-ddTHH:MM:SS+/-HH:MM'
		single-quoted string, (backslash to escape backslash or single quotes)
		true|false
		a field

	examples:
		mimeType = 'application/vnd.google-apps.folder'
		'someid' in parents
		'some@email' in owners
		'some@email' in writers
		sharedWithMe and name contains 'hello'

------------------
listing parameters
------------------
corpora: [user|domain|drive|allDrives]
driveId: id of ?shared? drive
includeItemsFromAllDrives: bool, include shared and MyDrive
orderBy: comma-delimited list of sort order criteria:
	[createdTime|folder|modifiedByMeTime|modifiedTime
	|name|name_natural|quotaBytesUsed|recency|sharedWithMeTime
	|starred|viewedByMeTime]
	Add 'desc' to sort descending instead of ascending.
	ex: '?orderBy=folder,modifiedTime desc,name'
pageSize: max number of items, max 1000, default 100
pageToken: string, get next page of a previous list command
q: string, query for filtering file results
spaces: string, comma-delimited list of spaces to query within the corpora
	[drive|appDataFolder]
supportsAllDrives: bool
includePermissionsForView: string, 'published'
includeLabels: string, comma-separated list of labels to put in labelInfo




listing response:
	{
		"files": [
			{objfile}
		],
		"nextPageToken": string,
		"kind": "drive#fileList",
		"incompleteSearch": bool
	}

scopes: one of:
	https://www.googleapis.com/auth/drive
	https://www.googleapis.com/auth/drive.appdata
	https://www.googleapis.com/auth/drive.file
	https://www.googleapis.com/auth/drive.meet.readonly
	https://www.googleapis.com/auth/drive.metadata
	https://www.googleapis.com/auth/drive.metadata.readonly
	https://www.googleapis.com/auth/drive.photos.readonly
	https://www.googleapis.com/auth/drive.readonly
