======================
-1.0- Addressing items
======================
2 parts:
	* the resource
	* the "facet"

url components:
	1. main url
	2. toplevel resource (?user?), "me", if authenticated
	3. drive to use
		drive: default drive
		drives/id: specify drive by id
	4. resource
		/root:path:
		/special/documents:path:
		/items/item-id

		NOTE: make sure to urlencode the path.
	6. facet
		/content

examples:
	https://graph.microsoft.com/v1.0/me/drive/root:/Documents/MyFile.xlsx:/content
	https://graph.microsoft.com/v1.0/me/drive/special/documents:MyFile.xlsx:/content
	https://graph.microsoft.com/v1.0/me/drive/items/0123456789AB/content
	https://graph.microsoft.com/v1.0/me/drives/AB0987654321/items/0123456789AB/content

===================
-1.1- authorization
===================
https://learn.microsoft.com/en-us/entra/identity-platform/v2-oauth2-auth-code-flow

register an app with "microsoft identity platform"
Generally tokens are valid for an hour.

Getting an auth token:
1. Get an auth code.
   Auth codes generally last 1 minute and are used to create an auth token.
   a. Direct user to https://login.microsoftonline.com/{tenant}/oauth2[/v2.0]/authorize
      depending on whether to use version 1.0 or 2.0.
   b. Add relevant query strings to control the login.
   c. User logs in
   d. Microsoft redirects user to the `redirect_uri` with extra query strings
      containing the auth code.
2. Use the auth code to get a token
   a. POST to ?corresponding domain?/oauth2/v2.0/token
      example:
      POST /{tenant}/oauth2/v2.0/token HTTP/1.1
      Host: https://login.microsoftonline.com
      Content-Type: application/x-www-form-urlencoded

      client_id=11112222-bbbb-3333-cccc-4444dddd5555
      &scope=https%3A%2F%2Fgraph.microsoft.com%2Fmail.read
      &code=OAAABAAAAiL9Kn2Z27UubvWFPbm0gLWQJVzCTE9UkP3pSx1aXxUjq3n8b2JRLk4OxVXr...
      &redirect_uri=http%3A%2F%2Flocalhost%2Fmyapp%2F
      &grant_type=authorization_code
      &code_verifier=ThisIsntRandomButItNeedsToBe43CharactersLong
3. Use the auth token to call apis.

Access can be delegated or app-only

* delegated
  user signs in. both user and app must be authorized.
  delegated permissions and user permissions must both
  allow the desired usage.
* app-only
  app acts on its own without any user (requires app certs)

tokens and codes are JWT
tokens can be validated for more security
NOTE: "public applications don't benefit from validation"
      ?? "malicious user can intercept/edit keys for validation"
When validating, it is recommended to check these values:
	timestamps
		iat ,nbf ,exp
	audience: aud should match app id
	nonce: the nonce during initial /authorize request.

=================
-1.1.1- auth code
=================
Getting an authcode requires some parameters:
====================
-1.1.1.1- parameters
====================
______
tenant
	required; the tenant to use: [common|organizations|consumers|tennant ids]
	This is the first part of the endpoint in the
	login.microsoft.com/{tenant}/... url
	Generally use `common`
_________
client_id
	required; the id for the registered application
	can use d50ca740-c83f-4d1b-b616-12c519384f0c (abraunegg's onedrive app id...)
_____________
response_type
	required; space-delimited string of [code|id_token|token]
____________
redirect_uri
	required; The redirect uri to send info to. It MUST have been registered and
	match exactly to a registered uri.
	typical options:
		https://login.microsoftonline.com/common/oauth2/nativeclient
		http://localhost
	It seems like after logging in from the browser, the user will be redirected
	to this `redirect_uri` with extra query params containing the token or
	something maybe...  Normally, the redirect_uri would be a link that the
	application controls.  When the application receives a GET http request, it
	knows that an authentication result is in the query strings. (or fragment
	or post, depending on response_mode)
_____
scope
	required; space-separated list of scopes. This seems to be X.permission
	examples:
		Files.ReadWrite[.All]
		Sites.ReadWrite[.All]
		offline_access
		openid (required to get an id token)
		profile
		email
_____________
response_mode
	recommended; The method to return response information.
	query: return as ?name=value&name=value... query string, default if only getting code
	fragment: return as #name=value&name=value..., default if getting id token
	form_post: use POST method, ?generally the most recommended?

	NOTE: fragment ?is safer? because it does not get sent to any servers, it
	remains in the browser only.
______
prompt
	optional
	type of user interaction:
	login: force login
	none: try using single-login, return "interaction_required" error if fail
	consent: triggers permission dialog
	select_account: choose an account
__________
login_hint
	optional; prespecify user/email, usually from `login_hint` JWT claim of previous sign-in
___________
domain_hint
	optional; similar to login_hint, but use `tid` JWT claim of previous sign-in
______________
code_challenge
	recommended; Client creates a "code_verifier" and sends a hash or some
	transformation.  The server records the final result and the method.  Client
	gets an access token, sending the original code_verifier as proof that the
	client is the original source of the request (server transforms code_verifier
	using the given method and it should match `code_challenge`)
_____________________
code_challenge_method
	recommended; plain(default)|S256, method to encode `code_verifier`
_____
state
	recommended; a value returned in the token response to prevent cross-site
	request forgery attacks In the response, it should be identical to the initial
	request.
_____
nonce
	required for an id token, inside the returned `id_token` as claim in JWT

==================
-1.1.1.2- response
==================
code=authorization_code
id_token=token
	contains a special `c_hash` JWT claim which is a hash of `code`.
state=state_from_parameters

====================
-1.1.2- access token
====================
=================
-1.1.2.1- request
=================
Access tokens are obtained with POST.  It uses nearly all the same parameters
as getting an authorization code.

Generally, move the querystring into the body and use
`content-type: application/x-www-form-urlencoded`

extra parameters:
____
code
	the authorization code.
__________
grant_type
	[authorization_code]
_____________
code_verifier
	The original of the `code_challenge` from getting the auth code.
_____________
client_secret
	Only for confidential apps.

_____________________
client_assertion_type
	'urn:ietf:params:oauth:client-assertion-type:jwt-bearer' to use a cert
________________
client_assertion
	JWT, signed with the registered cert for app-only mode.

==================
-1.1.2.2- response
==================
response is json
____________
access_token
	used for api
__________
token_type
	`Bearer` only
__________
expires_in
	duration til expiration in seconds
_____
scope
	nonstandard, default value is to use whatever the authorization code says.
_____________
refresh_token
	only if `offline_access` scope is requested.
________
id_token
	only if `openid` scope was requested.

===============
-1.1.3- refresh
===============
Tokens generally expire in 1 hour.  To continue using without re-login, refresh the token.
=================
-1.1.3.1- request
=================
Same as the initial POST to get the token except:
1. use `refresh_token` instead of `code`
2. no redirect_uri
3. grant_type=`refresh_token`
4. no code_verifier
==================
-1.1.3.2- response
==================
Response is identical to initial auth token response.
================
-1.1.3.3- errors
================
identical to initial errors.

Each item has "streams" that represent different aspects of the file (ex. thumbnail)
4 upload apis:

==========================
-1.2.1- simple item upload
==========================
Must be < 4mb
(but another sources says < 250mb)

permissions:
	permission type           least               most
	Delegated (work/school)   Files.ReadWrite     Files.ReadWrite.All, Sites.ReadWrite.All
	Delegated (personal)      Files.ReadWrite     Files.ReadWrite.All
	Application               Files.ReadWrite.All Sites.ReadWrite.All

-------
headers
-------
Authorization: "Bearer {token}"
Content-Type "text/plain. Required"

---------------
Replace an item
---------------
PUT /drives/{drive-id}/items/{item-id}/content
PUT /groups/{group-id}/drive/items/{item-id}/content
PUT /me/drive/items/{item-id}/content
PUT /sites/{site-id}/drive/items/{item-id}/content
PUT /users/{user-id}/drive/items/{item-id}/content

--------
new file
--------
PUT /drives/{drive-id}/items/{parent-id}:/{filename}:/content
PUT /groups/{group-id}/drive/items/{parent-id}:/{filename}:/content
PUT /me/drive/items/{parent-id}:/{filename}:/content
PUT /sites/{site-id}/drive/items/{parent-id}:/{filename}:/content
PUT /users/{user-id}/drive/items/{parent-id}:/{filename}:/content

resonse

-2- code_challenge
==================
1. create some random bytes
2. code_verifier = urlsafe_base64encode(randombytes).rstrip(b'=')
3. code_challenge is: urlsafe_base64encode(hashlib.sha256(code_verifier).digest()).rstrip(b'=')
